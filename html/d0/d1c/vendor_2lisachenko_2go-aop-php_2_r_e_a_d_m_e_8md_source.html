4 What is Go! AOP?
5 -----------------
6 
7 Go! AOP is a PHP 5.4 library based on OOP and AOP paradigms.
8 It allows developers to add support of AOP to every PHP application.
9 
10 Go! AOP doesn't require any PECL-extentions, it neither uses any dark magic of Runkit nor evals, the library doesn't use DI-containers.
11 The code with weaved aspects is fully readable and native, it can be easily debugged with XDebug.
12 You can debug either classes or aspects.
13 The main advantage of Go! AOP is that potentially it can be installed in every PHP-application,
14 because you don't have to change the application source code at all.
15 As an example, with the help of 10-20 lines of code we can intercept all the public, protected and static methods in all the classes
16 of application and display the name and the arguments of each method during its execution.

23 What is AOP?
24 ------------
25 
26 [AOP (Aspect-Oriented Programming)](http://en.wikipedia.org/wiki/Aspect-oriented_programming) is an approach to cross-cutting concerns, where the concerns are designed and implemented
27 in a "modular" way (that is, with appropriate encapsulation, lack of duplication, etc.), then integrated into all the relevant
28 execution points in a succinct and robust way, e.g. through declarative or programmatic means.
29 
30 In AOP terms, the execution points are called join points, a particular set of them is called a pointcut and the new
31 behavior that is executed before, after, or "around" a join point is called advice. You can read more about AOP in
32 [Introduction](http://go.aopphp.com/docs/introduction/) section.
33 
34 PHP traits can be used to implement some aspect-like functionality.
35 
36 Requirements
37 ------------
38 
39 Go! AOP library is developed for PHP 5.4.0 and up, but it can partially work with PHP 5.3.0 code.
40 
41 Note that PHP versions before 5.4 will not work completely, if you try to use
42  aspects for code that uses Late Static Binding (LSB) feature.
43 
44 Go! AOP library will not work with eAccelerator.

46 Known bugs
47 ------------
48 
49 * Version 5.4.5 of PHP has a [bug #62836 Seg fault or broken object references on unserialize()](https://bugs.php.net/bug.php?id=62836).
50 Class advices will be always broken after unserialize().
51 * Version 5.4.8 of PHP has a [bug #63481 Segmentation fault caused by unserialize()](https://bugs.php.net/bug.php?id=63481).
52 Interceptors for class can be unserialized incorrectly.
53 * Version 5.4.11 of PHP has a [bug #64070 Inheritance with Traits failed with error](https://bugs.php.net/bug.php?id=64070).
54 Method interception in traits is broken, however DeclareParent advice can be used correctly.

56 Installation
57 ------------
58 
59 Go! AOP library can be installed with composer or manually with git submodules. Installation is quite easy:
60 
61 1. Download the library using composer or git submodule
62 2. Create an application aspect kernel
63 3. Configure the aspect kernel in the front controller
64 4. Create an aspect
65 5. Register the aspect in the aspect kernel
66 
67 ### Step 1: Download the library using composer or git submodule
68 
69 Ask composer to download the Go! AOP library with its dependencies by running the command:
70 
71 ``` bash
72 $ php composer.phar require lisachenko/go-aop-php:0.4.0
73 ```
74 
75 Composer will install the library to your project's `vendor/lisachenko/go-aop-php` directory.
76 
77 Alternative way for installing the library is to use git submodules:
78 
79 ``` bash
80 git submodule add https://github.com/lisachenko/go-aop-php vendor/lisachenko/go-aop-php
81 ```
82 
83 Dependencies should be also installed as submodules:
84 ``` bash
85 git submodule add https://github.com/Andrewsville/PHP-Token-Reflection vendor/andrewsville/php-token-reflection
86 git submodule add https://github.com/doctrine/common vendor/doctrine/common (use 2.3.x tags)
87 git submodule add https://github.com/jakubledl/dissect vendor/jakubledl/dissect
88 ```

90 ### Step 2: Create an application aspect kernel
91 
92 The aim of this library is to provide easy AOP integration to your application.
93 Your first step then is to create the `AspectKernel` class
94 for your application. This class will manage all aspects of your
95 application in one place.
96 
97 The library provides base class to make it easier to create your own kernel.
98 To create your application kernel extend the abstract class `Go\Core\AspectKernel`
99 
100 ``` php
101 <?php
102 // app/ApplicationAspectKernel.php
103 
104 use Go\Core\AspectKernel;
105 use Go\Core\AspectContainer;
106 
110 class ApplicationAspectKernel extends AspectKernel
111 {
112 
120  protected function configureAop(AspectContainer $container)
121  {
122  }
123 }
124 ```
125 
126 ### 3. Configure the aspect kernel in the front controller
127 
128 To configure the aspect kernel, call `init()` method of kernel instance.
129 
130 ``` php
131 // front-controller, for Symfony2 application it's web/app_dev.php
132 
133 include __DIR__ . '/vendor/autoload.php'; // use composer
134 
135 // Initialize an application aspect container
136 $applicationAspectKernel = ApplicationAspectKernel::getInstance();
137 $applicationAspectKernel->init(array(
138  'debug' => true, // use 'false' for production mode
139  // Cache directory
140  'cacheDir' => __DIR__ . '/path/to/cache/for/aop',
141  // Include paths restricts the directories where aspects should be applied, or empty for all source files
142  'includePaths' => array(
143  __DIR__ . '/../src/'
144  )
145 ));
146 ```

148 ### 4. Create an aspect
149 
150 Aspect is the key element of AOP philosophy. And Go! AOP library just uses simple PHP classes for declaring aspects!
151 Therefore it's possible to use all features of OOP for aspect classes.
152 As an example let's intercept all the methods and display their names:
153 
154 ``` php
155 // Aspect/MonitorAspect.php
156 
157 namespace Aspect;
158 
159 use Go\Aop\Aspect;
160 use Go\Aop\Intercept\FieldAccess;
161 use Go\Aop\Intercept\MethodInvocation;
162 use Go\Lang\Annotation\After;
163 use Go\Lang\Annotation\Before;
164 use Go\Lang\Annotation\Around;
165 use Go\Lang\Annotation\Pointcut;
166 
170 class MonitorAspect implements Aspect
171 {
172 
179  public function beforeMethodExecution(MethodInvocation $invocation)
180  {
181  $obj = $invocation->getThis();
182  echo 'Calling Before Interceptor for method: ',
183  is_object($obj) ? get_class($obj) : $obj,
184  $invocation->getMethod()->isStatic() ? '::' : '->',
185  $invocation->getMethod()->getName(),
186  '()',
187  ' with arguments: ',
188  json_encode($invocation->getArguments()),
189  "<br>\n";
190  }
191 }
192 ```
193 
194 Easy, isn't it? We declared here that we want to install a hook before the execution of
195 all dynamic public methods in the class Example. This is done with the help of annotation
196 `@Before("execution(public Example->*(*))")`
197 Hooks can be of any types, you will see them later.
198 But we doesn't change any code in the class Example! I can feel you astonishment now )
199 
200 ### 5. Register the aspect in the aspect kernel
201 
202 To register the aspect just add an instance of it in the `configureAop()` method of the kernel:
203 
204 ``` php
205 // app/ApplicationAspectKernel.php
206 
207 use Aspect/MonitorAspect;
208 
209 //...
210 
211  protected function configureAop(AspectContainer $container)
212  {
213  $container->registerAspect(new MonitorAspect());
214  }
215 
216 //...
217 ```
218 
219 Now you are ready to use the power of aspects! Feel free to change anything everywhere.
